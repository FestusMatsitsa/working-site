# ICT Helpdesk & Asset Management System

An integrated ICT Helpdesk, Asset Management and Service Desk System for the Department of Lands, Energy, Physical Planning and Urban Development, Kilifi County Government.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080, exposed at /api)
- `pnpm --filter @workspace/helpdesk run dev` — run the frontend (port 21622, exposed at /)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite, Tailwind CSS, shadcn/ui, Recharts, wouter
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI contract (source of truth)
- `lib/db/src/schema/` — Drizzle ORM table definitions (users, assets, tickets, inventory, maintenance, knowledge)
- `artifacts/api-server/src/routes/` — Express route handlers (tickets, assets, inventory, maintenance, knowledge, users, dashboard)
- `artifacts/helpdesk/src/` — React frontend
- `lib/api-client-react/src/generated/` — Generated React Query hooks (do not edit)
- `lib/api-zod/src/generated/` — Generated Zod server schemas (do not edit)

## Architecture decisions

- OpenAPI-first: all API contracts defined in `openapi.yaml`; hooks and Zod schemas generated by Orval
- Single Express server at `/api` handles all backend routes
- Dashboard aggregation endpoints (summary, by-status, by-category, recent-tickets, etc.) computed in-server to avoid N+1 on the frontend
- Asset tags auto-generated as `ICT-XXXX` if not provided on creation
- Ticket comments use integer `is_internal` column (0/1) mapped to boolean at the API layer

## Product

- **Helpdesk** — Submit and track ICT support tickets by hardware, software, network, and user support categories
- **Asset Management** — Full ICT asset registry with QR-ready asset tags, assignment tracking, warranty and location management
- **Inventory** — Consumables and spare parts stock monitoring with low-stock alerts
- **Preventive Maintenance** — Schedule and track maintenance activities with overdue flagging
- **Knowledge Base** — Self-service articles for troubleshooting, procedures, policies, FAQs, and manuals
- **Dashboard & Reports** — Real-time summary metrics, ticket charts by status/category, upcoming maintenance, low-stock alerts

## Gotchas

- After any OpenAPI spec change, run `pnpm --filter @workspace/api-spec run codegen` before touching routes or frontend
- The API server must be restarted (`restart_workflow`) after code changes — it bundles at start
- Do NOT add leaf workspace packages to root `tsconfig.json` references (libs only)
- Body schema names in OpenAPI must be entity-shaped (e.g. `TicketInput`), not operation-shaped (`CreateTicketBody`), to avoid TS2308 collisions
